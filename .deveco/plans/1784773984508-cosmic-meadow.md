# Goal

在百花谷手机端（鸿蒙 + Android）和 WebUI 中，让设备的 DisplayName 更容易被看到和识别。当前问题：设备名仅在首次启动时设置一次，之后**再也不会在 UI 中出现**，用户无法确认或修改自己的设备名。

# Scope / Non-goals

**In scope:**
- 鸿蒙端：ProfilePage 显示设备名 + SettingsPage 关于区显示设备名/ID
- Android 端：Profile "我的"Tab 显示设备名
- 两端均可点击修改设备名

**Out of scope:**
- WebUI 改动（QRToolPanel 已有 inline edit）
- 改变 deviceId 生成逻辑（上轮已完成 AAID 改造）
- 后端 API 改动

# Current State And Constraints

## 现状
- 鸿蒙端 `DeviceNameStore.ets` 存储了用户自定义设备名（文化名如"听风笔"），通过 `getDeviceDisplayNameCustom()` 读取
- Android 端 `DeviceNameStore.kt` 同理，`getDeviceName(context)` / `getOrCreateDeviceName(context)` 读写
- 首次启动弹出"为你的花记取个名字"对话框 → 保存后**再不显示**
- ProfilePage 只显示华为账号名，没有设备名
- SettingsPage 关于区只有版本号和备案号，没有设备信息

## ArkTS 约束
- 不用 `any`，不用解构
- `async` 方法必须标注 `Promise<T>` 返回类型
- 对象字面量必须有类型上下文
- 权限：读取 preferences 不需要额外权限
- `@State` 变量初始化后可在 `aboutToAppear` 中异步更新

# Design

## 1. 鸿蒙端 ProfilePage — 头像卡片区增加设备名

**位置**: 现有头像+用户名卡片的用户名下方，添加一行设备名显示。

当前布局：
```
[📚头像]  [用户名 / 未登录]  [退出/登录按钮]
          [点数: N]
```

改为：
```
[📚头像]  [用户名 / 未登录]  [退出/登录按钮]
          [📱 听风笔]          ← 新增：设备名，可点击修改
          [点数: N]
```

- 设备名用 `📱` 前缀区分，`fontSize(13)`, `secondaryText` 色
- 点击设备名弹出修改对话框（复用 TextInput + 确定按钮的简单弹窗）
- 修改后调用 `setDeviceDisplayNameCustom()` 保存并更新 `@State`

## 2. 鸿蒙端 SettingsPage — 关于区增加设备信息

**位置**: 现有"关于"区块（版本 + 备案号），在版本行上方添加两行。

```
关于
┌────────────────────────────────┐
│ 设备名        听风笔           │  ← 新增
│ 设备标识      hm_a1b2c3d4...  │  ← 新增（截断显示，点击复制完整ID）
│ 版本          1.2.3           │
│ 备案号        豫ICP备...      │
└────────────────────────────────┘
```

- 设备名行：点击后可弹出修改对话框
- 设备标识行：显示 `getDeviceId()` 前 16 字符 + `...`，点击复制完整 ID 到剪贴板（已有 `pasteboard` 使用先例）
- 两行之间加分隔线

## 3. Android 端 "我的" Tab — 头像卡片区增加设备名

**位置**: 现有 `Card` 中头像+用户名的 `Column`，用户名下方增加设备名。

当前布局（Compose）：
```
[头像Icon]  [华为账号 / 用户名]  [退出/登录]
            [已登录 · xxx...]
```

改为：
```
[头像Icon]  [华为账号 / 用户名]  [退出/登录]
            [📱 听风笔]           ← 新增
            [已登录 · xxx...]
```

- 用 `DeviceNameStore.getDeviceName(context)` 读取
- `fontSize = 13.sp`, `color = onSurfaceVariant`
- 点击弹出修改对话框（`AlertDialog` + `OutlinedTextField`，与首次命名对话框风格一致）
- 修改后调用 `DeviceNameStore.setDeviceName()` 保存

## 4. WebUI — 无需改动

QRToolPanel 已支持 HostName（服务器名）inline 编辑。WebUI 是服务端，不是"设备端"，不显示"本设备名"。

# Risks And Compatibility

- **AAID 异步初始化时序**: SettingsPage 中 `getDeviceId()` 依赖 `initDeviceId()` 已完成（在 EntryAbility 启动时调用），需确认页面加载时缓存已就绪。fallback 机制兜底。
- **DeviceNameStore preferences key**: 鸿蒙端用 `huaji_device_name` preferences，Android 端用 `SharedPreferences`，读写接口不变，无需迁移。
